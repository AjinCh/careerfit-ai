# CareerFit AI Project Charter

## 1. Project Name

CareerFit AI

## 2. Project Mission

CareerFit AI helps international graduates and early-career professionals in Germany understand job market expectations, identify skill gaps, and improve their employability for data and AI roles.

The platform turns job descriptions and candidate profiles into explainable insights, practical recommendations, and learning priorities.

## 3. Problem Statement

Early-career candidates often face several challenges:

- Job descriptions are long, inconsistent, and difficult to compare
- Candidates are unsure which skills matter most for specific roles
- CVs may not clearly communicate relevant experience
- ATS readability is often misunderstood
- International graduates may not know how German job market expectations differ
- Beginners need structured guidance instead of vague advice

## 4. Target Users

CareerFit AI focuses on:

- International students and graduates in Germany
- Early-career data professionals
- Candidates applying for internships, working student roles, junior roles, or entry-level full-time roles
- Career switchers entering data and AI fields

## 5. Target Roles

Initial target roles:

- Data Analyst
- Data Scientist
- Machine Learning Engineer
- Applied AI Engineer
- Data Engineer
- MLOps Engineer
- GenAI Engineer

## 6. Product Positioning

CareerFit AI is a candidate-side self-improvement tool.

It helps candidates understand and improve their own readiness. It does not make hiring decisions and must not be used to automatically reject, rank, or filter candidates.

## 7. MVP Objective

The MVP should answer four core questions for a candidate:

1. What skills does this job require?
2. Which of these skills do I already have?
3. What are my most important gaps?
4. What should I learn or improve next?

## 8. Success Criteria for MVP

The MVP is successful if a user can:

- Paste or select a job description
- Enter their current skills
- See matched skills
- See missing skills
- Receive beginner-friendly recommendations
- Check whether their CV is likely to be ATS-readable
- Understand why recommendations were made

## 9. Non-Goals for Day 1

Day 1 will not include:

- Web scraping
- LLM integration
- Database setup
- User authentication
- Cloud deployment
- Advanced ML models
- Resume parsing
- Production security
- Employer-side candidate ranking
